# Milestone v1.8: Modulation Expansion + Arp/Looper Fixes

**Status:** ✅ SHIPPED 2026-03-07
**Phases:** 34, 35, 36, 37, 44
**Total Plans:** 7

## Overview

Extended the left-joystick modulation matrix to support cross-LFO targeting (X stick → LFO-Y params, Y stick → LFO-X params), expanded arp subdivisions to the full 17-item set matching the Random trigger system, allowed the arpeggiator to work with all trigger sources (not just TouchPlate), fixed the looper internalBeat_ double-scan bug causing phantom notes after overdub, and converted the plugin from MIDI effect type to VST3 instrument for universal DAW compatibility.

## Phases

### Phase 34: Cross-LFO Modulation Targets

**Goal**: Left Joystick X can target LFO-Y Frequency, Phase, and Level; Left Joystick Y can target LFO-X Frequency, Phase, and Level — extending the APVTS filterXMode/filterYMode choice params from 8 to 11 items each and adding the corresponding processBlock dispatch branches.
**Depends on**: Phase 33 (v1.7 shipped)
**Plans**: 2 plans

Plans:
- [x] 34-01-PLAN.md — PluginProcessor: extend APVTS to 11 items, cases 8/9/10 on both dispatch blocks, subdivMult guards, !liveGamepad sync reset
- [x] 34-02-PLAN.md — PluginEditor: 11-item ComboBox population + timerCallback cross-LFO visual tracking

**Key decisions:**
- Cross-LFO case 8 (Freq) mirrors existing case 4 pattern but reads APVTS from the opposite LFO — no new APVTS param IDs needed
- subdivMult reset guard extended: lfoXSubdivMult_ skips reset when yMode==8, symmetric for Y
- Cross-LFO slider tracking guards use the TARGET LFO playback state (not source axis state)
- New cross-LFO switch blocks inserted after (not inside) the existing yMode switch block

### Phase 37: Looper internalBeat_ Fix

**Goal**: Fix the LooperEngine double-scan bug where resetting internalBeat_ to 0.0 at beat boundaries causes beat-0 events to be scanned twice.
**Depends on**: Phase 36
**Plans**: 1 plan

Plans:
- [x] 37-01-PLAN.md — Remove internalBeat_=0.0 (line 773 in LooperEngine.cpp) + TC 14 regression test

**Key decisions:**

### Phase 44: Instrument Type Conversion (INSERTED)

**Goal**: Convert the plugin from isMidiEffect()=true to a VST3 instrument with a silent stereo audio output bus — achieving universal DAW compatibility while preserving all existing functionality.
**Depends on**: Phase 37
**Plans**: 2 plans

Plans:
- [x] 44-01-PLAN.md — CMakeLists + PluginProcessor: IS_MIDI_EFFECT FALSE, IS_SYNTH TRUE, output bus enabled
- [x] 44-02-PLAN.md — Build + DAW smoke test: Ableton instrument slot, Reaper stacked-track verified

**Key decisions:**
- IS_MIDI_EFFECT FALSE in CMakeLists required — JUCE VST3 wrapper ignores C++ isMidiEffect() for bus configuration
- Output bus enabled (true) in BusesProperties — instrument slot visibility requires active output bus
- .withInput() removed entirely — instruments don't consume audio input
- isBusesLayoutSupported accepts numOut=0 — DAWs may probe with zero outputs during discovery
- Ableton workflow: plugin must sit in instrument slot on Track 1; Track 2 reads MIDI from Track 1 output

## Milestone Summary

**Decimal/Out-of-sequence Phases:**
- Phase 44: Instrument Type Conversion (inserted into v1.8 from original v2.0 plan — fast win, no dependencies)

**Key Decisions (cross-phase):**
- filterXMode/filterYMode extended from 8 → 11 choices (indices 8/9/10 = cross-LFO targets)
- arpSubdiv APVTS param extended from 6 → 17 choices; breaking change accepted
- Force-TouchPlate arp guard removed — gate collision handled by existing isGateOpen check
- internalBeat_=0 sentinel removed — fmod absorbs overshoot correctly in both modes
- isMidiEffect() → false globally enables instrument slot placement

**Issues Resolved:**
- Looper phantom notes at boundary after overdub (TC 14 regression guard added)
- Arp locked to Pad mode even when Joystick/Random sources selected
- Arp rate limited to 6 options vs Random's 17
- Plugin invisible in instrument slot in Ableton, Cakewalk, FL Studio

**Technical Debt:**
- gamepad kStickBypassThreshold raised 0.05→0.10 (incidental fix, Phase 36 session) — may need tuning per controller
- Old arpSubdiv presets silently remap (not crash) — acceptable for pre-market beta
