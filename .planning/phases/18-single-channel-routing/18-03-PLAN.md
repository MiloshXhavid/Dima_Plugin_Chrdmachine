<task type="checkpoint:human-verify" gate="blocking">
  <what-built>
Single-Channel Routing feature:
- Routing panel below trigger section with "Routing:" mode dropdown (Multi-Channel / Single Channel)
- Multi-Channel mode: four per-voice channel dropdowns (Root ch, Third ch, Fifth ch, Tension ch)
- Single Channel mode: hides per-voice dropdowns, shows single "Channel" dropdown (1-16)
- Processor: effectiveChannel() routes all voices + filter CCs to the selected channel in Single mode
- noteCount_[16][128] deduplication: same-pitch collision → one noteOn, one noteOff
- Mode-switch and target-change flush allNotesOff before new routing takes effect
- Looper uses live channel setting at emission time (not record-time)
  </what-built>
  <how-to-verify>
Rescan the plugin in your DAW (Ableton Live). Open the plugin on a MIDI track routed to a MIDI monitor (e.g., Max for Live MIDI Monitor or the DAW's built-in MIDI output monitor).

**Test 1 — UI layout (ROUT-01):**
1. Open the plugin UI
2. Confirm: "Routing" panel is visible below the trigger/quantize section
3. Confirm: In Multi-Channel mode (default), four per-voice channel dropdowns are visible: Root ch, Third ch, Fifth ch, Tension ch
4. Switch dropdown to "Single Channel"
5. Confirm: Four per-voice dropdowns disappear; a single Channel dropdown (1-16) appears below the Routing dropdown
6. Switch back to "Multi-Channel" — per-voice dropdowns reappear

**Test 2 — Multi-Channel routing (ROUT-01 baseline):**
1. Leave routing in Multi-Channel mode
2. Press all four pads simultaneously
3. MIDI monitor must show: noteOns on channels 1, 2, 3, 4 respectively
4. No noteOns on any other channel

**Test 3 — Single Channel routing (ROUT-01, ROUT-02):**
1. Switch to Single Channel mode; set Channel to 5
2. Press all four pads simultaneously
3. MIDI monitor must show: all four noteOns on channel 5 only; no messages on channels 1-4

**Test 4 — Filter CC routing (ROUT-04):**
1. Still in Single Channel mode, channel 5
2. Move the left joystick left/right (or use gamepad left stick) to emit CC74
3. MIDI monitor must show: CC74 arriving on channel 5 only; no CC74 on ch1

**Test 5 — Note deduplication (ROUT-03):**
1. Still in Single Channel mode, channel 5
2. Set Root and Third voice channels to the same pitch (move joystick to a position where root and third produce the same pitch — e.g., third interval = 0)
3. Press both the Root pad and Third pad simultaneously
4. MIDI monitor must show: exactly ONE noteOn for that pitch on ch5; releasing both pads → exactly ONE noteOff
5. No doubled velocity, no stuck note

**Test 6 — Looper live channel (ROUT-05):**
1. Set Single Channel mode, channel 3
2. Record a short loop (press looper record, play some pads, stop record)
3. While looper is playing, change Channel dropdown to channel 7
4. MIDI monitor must show: looper notes now arriving on channel 7 (not channel 3)

**Test 7 — Mode-switch flush (no stuck notes):**
1. Hold a pad down in Multi-Channel mode
2. While holding, switch to Single Channel mode
3. Release the pad
4. No stuck notes — all notes cleared on mode switch

**Test 8 — Preset save/load:**
1. Set Single Channel mode, channel 9; voiceCh dropdowns set to custom values in Multi-Channel mode
2. Save preset; reload it
3. Routing mode and channel values should be restored correctly
  </how-to-verify>
