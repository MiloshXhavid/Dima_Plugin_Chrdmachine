# Phase 24.1 Plan 01: PluginProcessor Backend Refactor Summary

**Override-float + display-atomic architecture replaces writeParam lambda, preserving APVTS base values and enabling 30 Hz joystick visual tracking for LFO Rate/Phase/Level and Gate Length**

## Accomplishments

- Added 7 public `std::atomic<float>` display atomics to PluginProcessor.h (lfoXRateDisplay_, lfoXPhaseDisplay_, lfoXLevelDisplay_, lfoYRateDisplay_, lfoYPhaseDisplay_, lfoYLevelDisplay_, gateLengthDisplay_) — audio thread writes, timerCallback reads at 30 Hz
- Added 6 private audio-thread-only override floats (lfoXRateOverride_, lfoXPhaseOverride_, lfoXLevelOverride_, lfoYRateOverride_, lfoYPhaseOverride_, lfoYLevelOverride_) with -1.0f inactive sentinel; consumed by LFO params block with APVTS fallback
- Added private `juce::SmoothedValue<float, juce::ValueSmoothingTypes::Linear> smoothedGateLength_` for zipper-free gate modulation; initialized in prepareToPlay() with 50ms ramp
- Removed `writeParam` lambda that was destroying APVTS base values — replaced with base+offset model across all 6 LFO targets and gate length
- LFO params block (xp.rateHz, xp.phaseShift, xp.level; same for Y) now uses override floats with ternary APVTS fallback
- Gate length: SmoothedValue advances each processBlock; tp.gateLength and arp gateRatio both consume getCurrentValue()

## Decisions Made

- Base+offset model chosen over APVTS-write: stick center maps to slider base position, not mid-range. APVTS raw values are never overwritten.
- Override floats are plain `float` (audio-thread-only) rather than atomic — eliminates unnecessary synchronization overhead.
- Override floats reset unconditionally at top of dispatch block ensuring instant APVTS revert on mode change.
- Sync LFO Freq target retains existing lfoXSubdivMult_ exponential approach (no center-offset needed for subdivision).
