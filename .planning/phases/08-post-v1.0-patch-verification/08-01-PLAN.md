Current code:
```cpp

        ? ppqPos
        : looper_.getPlaybackBeat();

                  sampleOff);
```

Replace with:
```cpp
if (isOn)

        ? ppqPos
        : looper_.getPlaybackBeat();

    midi.addEvent(juce::MidiMessage::controllerEvent(ch0 + 1, 64, 127), sampleOff);
```

```cpp
if (loopOut.gateOn[v])
```

<task type="auto">
  <name>Task 2: PATCH-04 — Replace toggle-mute panic with one-shot wiring</name>
  <files>Source/PluginEditor.cpp</files>
  <action>
The panicBtn_ setup block starts around line 1078 in PluginEditor.cpp. The current wiring is wrong: it uses setClickingTogglesState(true) and calls proc_.setMidiMuted() creating a persistent MUTED state. The locked decision in STATE.md requires: "Panic is one-shot, not a mute toggle — press sends 48 CC events (16ch × CC64=0+CC120+CC123), button immediately returns to pressable."

Current code block (lines ~1081-1097):
```cpp
panicBtn_.setButtonText("MIDI PANIC");
panicBtn_.setClickingTogglesState(true);
panicBtn_.setTriggeredOnMouseDown(true);
panicBtn_.setTooltip("Toggle MIDI mute — kills all output. Press again to resume. Gamepad: R3.");
styleButton(panicBtn_);
panicBtn_.setColour(juce::TextButton::buttonColourId,  juce::Colour(0xFF5C1010));  // dark red (inactive)
panicBtn_.setColour(juce::TextButton::buttonOnColourId, Clr::highlight);            // bright red (muted)
panicBtn_.setColour(juce::TextButton::textColourOffId, Clr::text);
panicBtn_.setColour(juce::TextButton::textColourOnId,  juce::Colours::white);
panicBtn_.onClick = [this]
{
    const bool nowMuted = panicBtn_.getToggleState();
    proc_.setMidiMuted(nowMuted);
    if (nowMuted)
        proc_.triggerPanic();   // fires allNotesOff this block, then mute gate kicks in
    panicBtn_.setButtonText(nowMuted ? "MUTED" : "MIDI PANIC");
};
addAndMakeVisible(panicBtn_);
```

Replace the entire block with:
```cpp
panicBtn_.setButtonText("MIDI PANIC");
panicBtn_.setClickingTogglesState(false);   // PATCH-04: one-shot, not toggle
panicBtn_.setTriggeredOnMouseDown(true);
panicBtn_.setTooltip("Send allNotesOff on all voice channels and reset gates. Gamepad: R3.");
styleButton(panicBtn_);
panicBtn_.setColour(juce::TextButton::buttonColourId,  juce::Colour(0xFF5C1010));  // dark red
panicBtn_.setColour(juce::TextButton::textColourOffId, Clr::text);
panicBtn_.onClick = [this]
{
    proc_.triggerPanic();   // PATCH-04: one-shot — fires allNotesOff this block, button stays pressable
    // flashPanic_ is incremented from the audio thread; timerCallback handles the 167ms highlight
};
addAndMakeVisible(panicBtn_);
```

Key changes:
- setClickingTogglesState(false): button never enters a "toggled on" state
- Remove proc_.setMidiMuted() call entirely from the onClick — the UI button must NOT set midiMuted_
- Remove the buttonOnColourId colour (irrelevant now, no on-state)
- Remove the dynamic text change ("MUTED") — label stays "MIDI PANIC" always
- Remove nowMuted/getToggleState logic entirely
- Keep setTriggeredOnMouseDown(true) — fires on press not release, consistent with current UX

NOTE: The R3 gamepad toggle-mute path in processBlock (around line 379) remains UNCHANGED. That R3 → midiMuted_ toggle is separate functionality deferred to Phase 09. Phase 08 only fixes the UI button per PATCH-04.
