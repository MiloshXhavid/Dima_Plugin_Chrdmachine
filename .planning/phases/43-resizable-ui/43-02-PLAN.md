  <action>
Read PluginEditor::resized() in full. It begins around line 3953 (per RESEARCH.md sources). The existing first lines from Plan 01 are:
```cpp
scaleFactor_ = (float)getWidth() / 1120.0f;

```

Immediately after those two lines, define the sc() lambda:
```cpp
auto sc = [this](int px) -> int {
    return juce::jmax(1, juce::roundToInt((float)px * scaleFactor_));
};
```

Then systematically replace EVERY integer literal pixel constant in the resized() body with sc(n). This includes:
- `getLocalBounds().reduced(8)` → `.reduced(sc(8))`
- Every `area.removeFromTop(N)` → `area.removeFromTop(sc(N))`
- Every `area.removeFromBottom(N)` → `area.removeFromBottom(sc(N))`
- Every `area.removeFromLeft(N)` → `area.removeFromLeft(sc(N))`
- Every `area.removeFromRight(N)` → `area.removeFromRight(sc(N))`
- `constexpr int kLeftColW = 448` → use `sc(448)` where it appears
- LFO column widths (150px each) → `sc(150)`
- All row heights for LFO panel rows, arp rows, quantize rows, looper rows, gamepad area, touch plate rows, etc.
- setBounds() calls for all child components: the integer arguments become sc(n)
- `setBodyOffset(N)` for GamepadDisplayComponent → `setBodyOffset(sc(N))`
- Any explicit pixel values in `withWidth(N)`, `withHeight(N)`, `.translated(x, y)`, `.expanded(N)`, `.reduced(N)` within resized()
- Stored bounds rectangles (quantizerBoxBounds_, lfoXPanelBounds_, lfoYPanelBounds_, randomBoxBounds_, chordNameLabel_ bounds, etc.) all derive from the scaled area slices — no extra action needed as long as the removeFromTop/Left/Right/Bottom calls are all wrapped.

DO NOT wrap:
- `getWidth()`, `getHeight()`, `getLocalBounds()` — these are already in scaled pixels
- Proportional expressions like `area.getWidth() * 0.5f` — already proportional

After all layout constants are wrapped, add at the VERY END of resized() (after the last setBounds call):
```cpp
// Persist scale factor so getStateInformation can write it back
proc_.savedUiScale_.store((double)scaleFactor_);
```

In JoystickPad::resized() (inside PluginEditor.cpp), add at the end of the method body:
```cpp

displayCy_ = getHeight() * 0.5f - proc_.joystickY.load() * (getHeight() * 0.45f);
```

IMPORTANT: This is the largest single edit in the phase. Read resized() completely before editing. Work top-to-bottom through the function body. After the edit, build immediately to catch any missed constants or type errors.
