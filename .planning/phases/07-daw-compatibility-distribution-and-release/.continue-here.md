- Bug: MIDI notes held on DAW stop — `allNotesOff` only sent when `loopOut.dawStopped`,

  `const bool justStopped = (prevIsDawPlaying_ && !isDawPlaying) || loopOut.dawStopped;`
  Added `trigger_.resetAllGates()` on stop — clears pad/joystick/random gate state.
- Fix (PluginProcessor.h): Added `bool prevIsDawPlaying_ = false;` to private section.

<decisions_made>

- VST3_CATEGORIES "Fx" "MIDI Effect" (two CMake list items) produces clean "Fx|MIDI Effect".
  Single string "Fx|MIDI Effect" produces "Fx|Fx|MIDI Effect" — CMake list concatenation bug.

- Ableton 11.3.43 classifies "Fx|MIDI Effect" as `audiofx` (not `midifx`). Requires audio buses.
  Workaround: declare optional inactive stereo buses — Ableton satisfies its bus queries then
  deactivates them for MIDI track use. processBlock already calls audio.clear() so passthrough is silent.

- isMidiEffect()=true kept. JUCE VST3 wrapper reads bus count from BusesProperties, not isMidiEffect().

- DAW stop note-off: detect transition at PluginProcessor level (not inside LooperEngine) so it
  covers both sync-on and sync-off cases. Use `prevIsDawPlaying_` bool member on audio thread.

- `trigger_.resetAllGates()` must be called on DAW stop (same as processBlockBypassed does)
  to prevent stale gate state from leaking into next session.
</decisions_made>
