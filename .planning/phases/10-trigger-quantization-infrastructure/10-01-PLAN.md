  <name>snapToGrid — nearest-grid-boundary pure function with wrap guard</name>
  <files>Source/LooperEngine.h, Source/LooperEngine.cpp, Tests/LooperEngineTests.cpp</files>
  <behavior>
    Pure free function. Takes (beatPos, gridSize, loopLen). Returns nearest grid boundary, always in [0, loopLen) via std::fmod.

    Tie-breaking rule (LOCKED — from CONTEXT.md):
    - Ties snap to the EARLIER grid point (beatPos strictly less than midpoint → lower; else upper)
    - This avoids "jumped a beat" feeling when a gate fires exactly at the midpoint between two grid lines

    Wrap guard rule (LOCKED — from STATE.md):
    - std::fmod(snapped, loopLen) is MANDATORY — if snapped == loopLen, it wraps to 0.0
    - IEEE 754: std::fmod(x, x) == 0.0 for finite positive x — this is guaranteed

    Edge cases (all tested in TC 12):
    - beatPos = 3.999, gridSize = 1.0, loopLen = 4.0 → nearest grid = 4.0 → fmod → 0.0
    - beatPos = 3.5,   gridSize = 1.0, loopLen = 4.0 → exact midpoint between 3.0 and 4.0 → ties snap earlier → 3.0 (NOT 4.0)
    - beatPos = 0.001, gridSize = 1.0, loopLen = 4.0 → nearest = 0.0
    - beatPos = 2.3,   gridSize = 0.5, loopLen = 4.0 → lower=2.0, upper=2.5, mid=2.25 → 2.3 >= 2.25 → 2.5
    - gridSize <= 0 or loopLen <= 0 → return beatPos unchanged (guard)

    quantizeSubdivToGridSize conversion (also add here):
    - 0 → 1.0 (1/4 note)
    - 1 → 0.5 (1/8 note)
    - 2 → 0.25 (1/16 note)
    - 3 → 0.125 (1/32 note)
    - default → 0.5 (safe fallback per research)
  </behavior>

    // Ties (beatPos exactly at grid midpoint) snap to the EARLIER grid point.

    double snapToGrid(double beatPos, double gridSize, double loopLen) noexcept;

    // Converts quantizeSubdiv APVTS index (0..3) to beat-unit grid size.

    STEP 2 — Implement in LooperEngine.cpp (add at the top, after includes, before class methods):

    ```cpp
    double snapToGrid(double beatPos, double gridSize, double loopLen) noexcept
    {
        if (gridSize <= 0.0 || loopLen <= 0.0) return beatPos;
        const double lower    = std::floor(beatPos / gridSize) * gridSize;
        const double upper    = lower + gridSize;
        const double midpoint = lower + gridSize * 0.5;
        // Ties go to earlier grid point: use strict less-than so beatPos == midpoint → upper
        // Wait — CONTEXT.md says ties snap to EARLIER. Test case: beatPos=3.5, mid=3.5 → earlier=3.0.
        // So: beatPos >= midpoint → snap to upper ONLY when beatPos is closer to upper.
        // beatPos == midpoint → not strictly less than midpoint → snap to lower (earlier).
        // Correct condition: (beatPos < midpoint) ? lower : upper   BUT this gives upper for ==
        // Fix: use (beatPos <= midpoint) ? lower : upper for ties-to-earlier behavior
        const double snapped  = (beatPos <= midpoint) ? lower : upper;
        return std::fmod(snapped, loopLen);
    }

    double quantizeSubdivToGridSize(int subdivIdx) noexcept
    {
        switch (subdivIdx)
        {
            case 0: return 1.0;
            case 1: return 0.5;
            case 2: return 0.25;
            case 3: return 0.125;
            default: return 0.5;
        }
    }
    ```

    IMPORTANT tie-breaking note: CONTEXT.md says "ties snap to EARLIER" meaning beatPos == midpoint should return lower (earlier). The correct comparison is `beatPos <= midpoint` → lower, `beatPos > midpoint` → upper. This differs from the RESEARCH.md code example which uses `beatPos < midpoint` (that snaps ties to UPPER). The CONTEXT.md locked decision overrides the research example — use `<=`.
